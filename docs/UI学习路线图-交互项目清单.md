# UI 学习路线图（交互项目清单）

目标：把 UI 设计从底层能力拆成可交互、可反复练习的主题项目；每个项目都能在网页中即时反馈，并能复用同一套 UI/状态/规范，长期维护也不走样。

## 总体原则（避免做成碎片化笔记）

- **每个项目=一个可交互工具**：输入/调参 → 实时结果 → 给结论与建议（为什么、怎么改）。
- **优先 2D**：贴近真实 UI，成本更低；**Three.js 只在“空间化理解”明显加分时使用**。
- **每个项目必须有“可复用产物”**：颜色 token、排版 token、组件状态、布局参数、检查报告等（可复制/导出）。
- **统一交互结构**（后续复用）：左侧目录选择主题；中间内容可视化；右侧面板负责参数与说明；顶栏放“随机/重置/导出/上传”等动作。

## 01 排版与可读性（Typography）

学习目标：把“看起来舒服/高级”的原因说清楚，并能通过参数稳定复现。

### 项目 A：层级系统练习（标题/正文/注释）
- **内容区**：一段真实 UI 文案示例（标题 + 列表 + 表单说明 + 错误提示）。
- **面板交互**：
  - 字号：`标题/正文/注释`（独立或比例联动）
  - 行高、字重、字距（Letter spacing）
  - 段间距与列表间距（对齐 tokens）
  - “推荐档位”一键切换（例如：紧凑/舒适/阅读）
- **反馈方式**：
  - 给出“阅读舒适度提示”（行长区间、行高建议、层级对比是否足够）
  - 标记层级冲突（标题与正文差异不足）
- **可复用产物**：Typography tokens（字号/行高/字重/字距/段落间距）一键复制为 CSS 变量片段。

### 项目 B：行长 × 行高舒适区
- **内容区**：同一段文本三列对比（当前 vs 推荐1 vs 推荐2）。
- **面板交互**：容器宽度（行长）、行高、字号、字体选择（系统字体即可）。
- **反馈方式**：高亮“舒适区”范围；提示为什么太挤/太散。

## 02 色彩与可用性（Color for UI）

学习目标：颜色不是“好看”，而是“可用 + 一致 + 可解释”。

### 项目 A：对比度/WCAG 检查器（推荐优先做）
- **内容区**：模拟 UI 卡片（标题/正文/按钮/链接）实时预览。
- **面板交互**：
  - 前景/背景色（支持输入 hex + 取色 + 互换）
  - 文本大小与字重（影响 WCAG 级别）
  - 场景切换：正文/按钮/链接/禁用态
- **反馈方式**：
  - AA/AAA 是否通过（大字/小字分别判断）
  - 不通过给出“改哪一个更划算”：提高明度差、降低饱和、调整 alpha 等
  - 一键生成“通过版”颜色（保留原色相倾向）
- **可复用产物**：颜色 token（primary/bg/text/muted/border）与通过报告。

### 项目 B：状态色系统（Success/Warning/Error/Info）
- **内容区**：同一套组件在 4 种语义色下的展示（badge/alert/button）。
- **面板交互**：
  - 深色/浅色背景切换
  - 语义色的“强度”（更偏 UI 还是更偏插画）
- **反馈方式**：提示“信息层级”是否清晰（文本/边框/底色对比）。

### 项目 C（可选）：色觉模拟（学习向）
- **内容区**：在不同色觉模式下预览（正常/红绿色弱等）。
- **面板交互**：模式选择 + 强度。
- **反馈方式**：提示哪些信息仅靠颜色传达，建议加形状/文本冗余。

## 03 布局与网格（Layout System）

学习目标：理解“对齐/间距/节奏”是可度量的系统，而不是感觉。

### 项目 A：8pt 间距系统练习
- **内容区**：一个可拖拽的小页面骨架（卡片、按钮、标题、列表）。
- **面板交互**：
  - 间距 token（8/12/16）与组合
  - 圆角 token（12/16）
  - “自动对齐/自动吸附”开关
- **反馈方式**：
  - 标记哪些间距不是 token 倍数（提示“走样点”）
  - 一键“规范化”（把间距修到最近 token）
- **可复用产物**：页面骨架参数（可导出 JSON 或 CSS 变量片段）。

### 项目 B：栅格系统（12 列 + 断点）
- **内容区**：拖拽模块（hero、列表、侧栏）到栅格上；随宽度改变响应式。
- **面板交互**：列数、栏距、边距、断点预设。
- **反馈方式**：提示对齐/溢出/不合理的跨列比例。

> Three.js 选择：一般不需要。若想做“空间化理解”，可把栅格/组件/约束关系做成可旋转分层，但优先放到后期。

## 04 组件与状态（Components & States）

学习目标：组件不是静态图；状态、可访问性与一致性是关键。

### 项目 A：组件状态矩阵（强推荐）
- **内容区**：Button / Input / Select / Toggle 的状态矩阵（default/hover/active/focus/disabled/loading）。
- **面板交互**：
  - 组件选择、尺寸（S/M/L）
  - 状态切换（或自动演示）
  - 焦点可见性（focus ring）开关与强度
- **反馈方式**：
  - 提示状态差异是否足够（hover 不明显、disabled 太像默认）
  - 提示无障碍问题（focus 不可见、对比不足）
- **可复用产物**：组件 tokens（高度、padding、radius、stroke、focus ring）。

### 项目 B：表单信息结构（Label/Help/Error）
- **内容区**：一组表单示例，演示正确的信息层级与错误呈现。
- **面板交互**：错误文案长度、帮助文案开关、必填标记样式。
- **反馈方式**：提示“可扫描性”与“错误定位成本”。

## 05 交互与动效（Motion）

学习目标：动效是信息层级与反馈，不是装饰。

### 项目 A：Easing 与时长对比（克制）
- **内容区**：同一交互（展开/切换/提示）在不同曲线与时长下对比。
- **面板交互**：时长、曲线、位移距离、是否使用模糊/阴影。
- **反馈方式**：提示“过慢/过快/过飘”的原因与推荐区间。

> Three.js 选择：基本不需要；2D 曲线/时间轴足够清晰。

## 06 设计系统（Tokens & Theming）

学习目标：把前面的结论沉淀为可复用系统，减少走样。

### 项目 A：Token 浏览器（全站）
- **内容区**：展示当前 tokens（spacing/radius/stroke/type/color）以及使用示例。
- **面板交互**：主题切换（深/浅）、密度（紧凑/舒适）、圆角风格（12/16）。
- **可复用产物**：一键导出当前 tokens。

### 项目 B：主题切换（Dark/Light）与对比度保护
- **内容区**：同一套组件在深/浅主题切换，观察可用性是否保持。
- **面板交互**：背景亮度、文本层级强度、描边强度。

## 项目落地到代码结构（与当前工程一致）

约定（不增加复杂度）：
- 新分类（Project） = `src/registry.js` 增加一个 `project`（例如 `typography`/`layout`/`components`）。
- 新项目（Topic） = `src/projects/<projectId>/topics/<topicId>/topic.js` 新增一个目录/文件。
- 主题可复用基础设施 = `window.DesignBookKit`（见 `src/designbook-kit.js`）。
- 3D 主题（可选）在 `registry.js` 标记 `needsThree: true`。
- UI tokens 统一见 `src/styles.css`，并参考 `docs/UI-规范.md`。

## 推荐的“下一步最小路线图”（避免返工）

1. 先做：**对比度/WCAG 检查器**（色彩分类内），它会反哺全站颜色 tokens 与可用性规范。
2. 再做：**层级系统练习**（排版分类内），建立 Typography tokens。
3. 再做：**组件状态矩阵**（组件分类内），把 tokens 真正落地成组件规范。
